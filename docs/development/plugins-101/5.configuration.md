# 插件配置

IPE 使用 [Schemastery](https://github.com/nicot/schemastery) 来定义插件的配置项。配置项会自动出现在"首选项"面板中，用户可以在那里修改。

## 注册配置

最简单的方式是通过 `PreferencesSchema` 声明：

```ts twoslash
import { Schema } from '@inpageedit/core'

ipe.plugin({
  inject: ['preferences'],
  PreferencesSchema: Schema.object({
    myPlugin: Schema.object({
      greeting: Schema.string().default('hello'),
      enabled: Schema.boolean().default(true),
    }),
  }).extra('category', 'general'),
  apply(ctx) {
    // read preferences
    const greeting = await ctx.preferences.get('myPlugin.greeting')
  },
})
```

如果你使用 `BasePlugin` + 装饰器，可以用 `@RegisterPreferences`：

```ts twoslash
import { BasePlugin, Inject, RegisterPreferences, Schema } from '@inpageedit/core'

@Inject(['preferences'])
@RegisterPreferences(
  Schema.object({
    myPlugin: Schema.object({
      greeting: Schema.string().default('hello'),
      enabled: Schema.boolean().default(true),
    }),
  }).extra('category', 'general')
)
export default class MyPlugin extends BasePlugin {
  // ...
}
```

::: tip

Schema 只需要定义 **结构和默认值**，不需要调用 `.description()`。描述文本由 i18n 文件提供，详见下方的 [i18n 约定](#i18n-约定)。

:::

## Schema 常用写法

```ts twoslash
import { Schema } from '@inpageedit/core'

Schema.object({
  myPlugin: Schema.object({
    // boolean
    enabled: Schema.boolean().default(true),

    // string
    summary: Schema.string().default('default text'),

    // union (select)
    mode: Schema.union([
      Schema.const('auto'),
      Schema.const('manual'),
      Schema.const('disabled'),
    ]).default('auto'),

    // keyboard shortcut
    keyshortcut: Schema.string().default('ctrl-s').role('keyshortcut'),
  }),
}).extra('category', 'general')
```

### `.extra('category', name)`

指定配置项出现在哪个分类 tab 下。内置分类有 `general`、`editor`。

### `.default(value)`

设置默认值，**必须提供**。

### `.role(name)`

为字段指定特殊的渲染角色。例如 `'keyshortcut'` 会渲染为快捷键录制器。

### `.extra('hidden', true)`

隐藏字段，不在首选项面板中显示，但仍可通过代码读写。

## 读取和监听配置

```ts twoslash
// read
const value = await ctx.preferences.get('myPlugin.enabled')

// listen for changes
ctx.on('preferences/changed', (e) => {
  if (e.key === 'myPlugin.enabled') {
    console.log('enabled changed to', e.value)
  }
})
```

## i18n 约定 {#i18n-约定}

配置项的描述文本**不写在 Schema 中**，而是由 i18n 文件通过约定式 key 自动解析。

### Key 规则

| 用途                 | Key 格式                        | 示例                                            |
| -------------------- | ------------------------------- | ----------------------------------------------- |
| 组标题               | `prefs.{group}.$title`          | `prefs.myPlugin.$title` → "My Plugin Settings"  |
| 字段描述（叶子节点） | `prefs.{group}.{field}`         | `prefs.myPlugin.greeting` → "Greeting message"  |
| 字段描述（有子节点） | `prefs.{group}.{field}.$label`  | `prefs.myPlugin.mode.$label` → "Operation mode" |
| Union 选项文本       | `prefs.{group}.{field}.{value}` | `prefs.myPlugin.mode.auto` → "Automatic"        |
| 分类 tab 名称        | `prefs.$category.{name}`        | `prefs.$category.general` → "General"           |

### YAML 示例

对应上面 Schema 的 i18n YAML：

```yaml
prefs:
  $category:
    general: General
  myPlugin:
    $title: My Plugin Settings
    greeting: Greeting message
    enabled: Enable my plugin
    mode:
      $label: Operation mode
      auto: Automatic
      manual: Manual
      disabled: Disabled
```

### 简单字段 vs 有子节点的字段

如果一个字段是叶子节点（boolean、string 等），直接用裸 key：

```yaml
prefs:
  myPlugin:
    enabled: Enable my plugin # leaf node, bare key
```

如果一个字段同时需要自身描述和子节点（如 union 选项），用 `$label`：

```yaml
prefs:
  myPlugin:
    mode: # has children
      $label: Operation mode # self description
      auto: Automatic # option label
      manual: Manual
      disabled: Disabled
```

这是因为 YAML 中一个 key 不能同时是标量和对象。

::: warning

不要在 Schema 上调用 `.description()`。所有描述文本都应通过 i18n 文件提供。

:::
